# Certification Evaluation Framework

This framework provides an objective way to evaluate if a certification is worth pursuing. We evaluate on a 25-point scale. Only certifications scoring ≥20/25 should be included in the roadmap.

## Criteria (5 points each)

### 1. Market Weight (Peso en el mercado)
Does the industry ask for this? Is it recognized by hiring managers and tech leads?
- `1`: Unknown or vendor-locked without broad adoption.
- `3`: Good to have, but not a hard requirement.
- `5`: Industry standard (e.g., CKA, AWS SAA).

### 2. Cost (Costo)
Is the financial investment reasonable for the expected return?
- `1`: Extremely expensive (>$1000) with no employer coverage.
- `3`: Moderately expensive ($200-$400).
- `5`: Free or <$100.

### 3. Preparation Time (Tiempo de preparación)
How much time does it take away from building real projects?
- `1`: 6+ months of pure theoretical study.
- `3`: 2-3 months.
- `5`: <1 month, especially if the knowledge is already used daily.

### 4. Alignment with Current Work (Alineación con mi trabajo)
Can I apply this immediately in my day-to-day work?
- `1`: Completely theoretical or unused stack.
- `3`: Occasionally used or planned for future quarters.
- `5`: Used daily in production.

### 5. Alignment with Career Goal (Alineación con mi objetivo)
Does this directly build the "Senior DevOps / Platform Engineer" profile?
- `1`: Irrelevant.
- `3`: Tangential (e.g., frontend/backend specifics).
- `5`: Core Platform/SRE skill.

---

## Example Evaluation: Octopus Fundamentals

- Market Weight: 2/5 (Niche compared to ArgoCD/GitHub Actions)
- Cost: 4/5 ($49.95 is accessible)
- Preparation Time: 4/5 (Fast)
- Alignment with Work: 1/5 (Not used in current production stack)
- Alignment with Goal: 3/5 (CI/CD is relevant, but the tool isn't priority)
- **Total: 14/25 (Rejected - <20)**
